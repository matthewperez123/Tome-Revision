# Tome Luminous Minimal Modernism

This note records the June 18 lighter-cover direction used for corpus-wide Tome cover production.

## Active References

- `docs/cover-system/references/luminous-june-18/luminous-minimal-suite-a-2026-06-18-161222.png`
- `docs/cover-system/references/luminous-june-18/luminous-minimal-suite-b-2026-06-18-161341.png`

Suite A is the 4:12 Plato/Republic-left reference: garden approaches, civic geometry, columns, open doors, keys, trees, and daylight philosophical space.

Suite B is the 4:13 first-example reference: shore, vessel, harbor, causeway, island, open horizon, and voyage vectors.

The rightward 4:15 suite and darker gothic/navy suites are not active style sources for this mode.

## Difference From The Darker Suites

- The atmosphere is light-first: ivory, sage, sea glass, wheat, celadon, clay rose, clear sky, warm stone, orchard mist, and pale snow.
- Deep ink is reserved for linework, silhouettes, and small accents; it does not set the whole cover mood.
- Philosophical works prefer civic daylight, columns, plazas, open thresholds, and public axes rather than default cave darkness.
- Gothic or tragic books are lightened through snow, room-edge, threshold, or isolated-dwelling families instead of horror-poster lighting.
- Every cover should be book-specific, but the subject is reduced to one symbol and one vector.

## Full-Corpus Rollout

1. Analyze the two active June 18 references with `npm run covers:analyze-references`.
2. Run Pilot A in deterministic mock mode and review the contact sheet.
3. Adjust palette/composition weighting if more than 15 percent of the pilot feels dark, crowded, repetitive, cropped, or non-luminous.
4. Run Pilot B for Standard Ebooks middle-content conversion where source clues exist.
5. Run 3 to 5 additional mock batches of 16 books across genre bands.
6. Validate Imagen credentials with `npm run covers:validate-imagen`.
7. Request explicit approval for exactly one paid Imagen proof book.
8. Review raw 3:4, Tome 2:3 crop, derivatives, QA, prompt hash, and duplicate warnings.
9. Continue paid batches of 8, then 16 only after multiple batches pass review.
10. Pause on provider failure, budget cap, credentials warning, style drift, duplicate motif clustering, or app integration failure.

No paid generation is allowed until the one-book proof is explicitly approved.
